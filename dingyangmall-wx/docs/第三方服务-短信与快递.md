# 第三方服务接入说明（短信 + 快递查询）

## 一、腾讯云短信

### 1. 依赖

- 父 POM：`pom.xml` 的 `dependencyManagement` 中已声明 `tencentcloud-sdk-java-sms`。
- 使用模块：`dingyangmall-framework/pom.xml` 中已引入该依赖。

### 2. 配置类

- `dingyangmall-framework/src/main/java/com/dingyangmall/framework/config/TencentSmsProperties.java`
  - 读取 `tencent.sms.*` 配置项。

### 3. 业务代码

- `dingyangmall-framework/src/main/java/com/dingyangmall/framework/web/service/SmsService.java`
  - `sendSmsCode(phone)`：生成 6 位验证码写入 Redis（5 分钟有效），若 `tencent.sms.enabled=true` 且已配置 SecretId，则调用腾讯云短信 API 发送。
  - `validateSmsCode(phone, code)`：校验验证码并从 Redis 删除。

### 4. 配置示例（application.yml 或环境变量）

```yaml
tencent:
  sms:
    enabled: true
    secret-id: 你的SecretId
    secret-key: 你的SecretKey
    sms-sdk-app-id: 你的SdkAppId
    sign-name: 你的签名
    template-id: 你的验证码模板ID
    region: ap-guangzhou
```

未配置或 `enabled: false` 时，仅模拟发送（验证码只存 Redis，不真实发短信）。

## 二、快递100（Api100）物流查询

### 1. 实现位置（无新增 Maven 依赖，使用 Hutool HTTP）

- 配置：`dingyangmall-mall/.../config/MallConfigProperties.java` 增加 `kuaidi100Key`、`kuaidi100Customer`。
- 接口：`dingyangmall-mall/.../service/Kuaidi100QueryService.java`
- 实现：`dingyangmall-mall/.../service/impl/Kuaidi100QueryServiceImpl.java`
  - 请求：`POST https://poll.kuaidi100.com/poll/query.do`，表单参数 `customer`、`sign`、`param`（param 为 JSON，含 com、num 等）；签名规则：`MD5(param + key + customer)` 转 32 位大写。
- DTO：`dingyangmall-mall/.../dto/ExpressTrackResult.java`（含轨迹列表 `data`）。

### 2. 调用处

- `dingyangmall-admin/.../web/api/OrderInfoApi.java` 的 `getLogistics(id)`：
  - 根据订单 ID 取订单与订单物流，再调用 `Kuaidi100QueryService.query(orderLogistics)` 得到实时轨迹。
  - 返回：`{ "logistics": 订单物流实体, "track": ExpressTrackResult }`。

### 3. 配置示例（application.yml 的 mall 下）

```yaml
mall:
  notify-host: https://你的域名
  kuaidi100-key: 你的Key
  kuaidi100-customer: 你的Customer
```

未配置 key/customer 时，物流接口仍可访问，但 `track` 仅含本地物流字段，无实时轨迹列表。

### 4. 快递公司编码

- 与系统已有 `OrderLogisticsEnum` 一致（如 yuantong、shunfeng、ems、zhongtong 等），直接传给快递100 的 `com` 参数即可。

## 三、接口汇总

| 能力       | 接口路径（示例）                         | 说明 |
|------------|------------------------------------------|------|
| 发送验证码 | `GET /common/sms/send?phone=xxx`         | 管理端 |
| 发送验证码 | `GET /app/member/send-sms-code?phone=xxx` | 移动端 |
| 物流+轨迹  | `GET /weixin/api/ma/orderinfo/logistics/{订单id}` | 移动端，需登录，返回 logistics + track |

以上内容与项目 README 中的「第三方服务配置」一节一致，便于开发与运维对照。
