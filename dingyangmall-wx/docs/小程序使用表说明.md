# 小程序端使用的数据库表（仅读/写已有表）

以下为小程序相关接口实际用到的表，**全部为现有表，未新增任何表**。

## 小程序请求与 goods_spu 表对应关系（查询所有商品）

| 小程序端请求 | 后端接口 | 是否查 goods_spu 表 |
|--------------|----------|----------------------|
| `api.getHomePage()` | GET /api/ma/home | ✅ 是（首页轮播、猜你喜欢、推荐块均查 goods_spu） |
| `api.goodsPage(data)` | GET /api/ma/goodsspu/page | ✅ 是（分页查 goods_spu，shelf=1） |
| `api.goodsListAll(limit)` | GET /api/ma/goodsspu/list | ✅ 是（直接查所有上架商品，默认最多 500 条） |
| `api.goodsGet(id)` | GET /api/ma/goodsspu/{id} | ✅ 是（按 id 查 goods_spu） |

后端查询逻辑：`GoodsSpuApi` / `ApiMaHomeController` → `GoodsSpuService.page1()` 或 `getById2()` → `GoodsSpuMapper` → SQL **FROM goods_spu**。

## 其他功能与表

| 功能           | 使用表            | 说明                     |
|----------------|-------------------|--------------------------|
| 分类树/宫格     | `goods_category`  | 商品分类                 |
| 公告           | `sys_notice`      | 系统公告                 |
| 小程序用户/登录/手机号 | `wx_user`     | 旧表，open_id 维度读写   |
| 订单           | `order_info`、`order_item`、`order_logistics` | 订单相关 |
| 购物车         | `shopping_cart`   | 购物车                   |
| 收货地址       | `user_address`    | 用户地址                 |

- 未使用 `wx_ma_user` 做小程序用户持久化，统一使用 `wx_user`。
- `sql/goods_display_check.sql` 仅含 SELECT/UPDATE，**无 CREATE TABLE**，用于自检商品是否上架。

新功能如需新表再单独建表；现有小程序逻辑不增加新表。
