# 管理端 - 人员与会员接口清单

便于前端改管理端时直接对照的接口说明。**四个都要**：一级经销商、二级经销商、终端客户、小程序用户；二级经销商需标明从属关系。

---

## 一、会员管理（小程序用户 + 终端会员）

数据来源：**以 wx_user 为主表**，按手机号关联 ums_member，带积分、余额、等级、会员码。

| 功能 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 会员分页 | GET | `/mall/member/page` | `current`(默认1)、`size`(默认10)，或 `pageNum`/`pageSize`；可选 `phone`、`nickname` 模糊 | 每条含：id, openid, nickname, avatarUrl, phone, createTime, updateTime, memberId, points, balance, level, memberCode |
| 会员详情 | GET | `/mall/member/{id}` | 路径 id：优先 wx_user 主键，否则 ums_member.id | 同上字段；若按 ums_member 查则多返回 wxOpenid、wxAvatarUrl、wxNickname |
| 会员新增 | POST | `/mall/member` | body: UmsMember | - |
| 会员编辑 | PUT | `/mall/member` | body: UmsMember | - |
| 会员删除 | DELETE | `/mall/member/{id}` | 路径 id | - |

---

## 二、用户/账户管理（一级、二级经销商）

数据来源：**sys_user**。二级经销商通过 `parentDistributorId`、`parentDistributorName` 标明从属关系。

| 功能 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 用户列表 | GET | `/system/user/list` | `pageNum`、`pageSize`；可选 `userName`、`phonenumber`、`status`、`dealerLevel`(1=一级 2=二级)、`parentDistributorId` | 每条含：userId, userName, nickName, phonenumber, status, **dealerLevel**, **parentDistributorId**, **parentDistributorName**(从属关系), deptId, createTime |
| 用户详情 | GET | `/system/user/{userId}` | 路径 userId | 返回 user 对象 + parentDistributorName（二级时） |

---

## 三、前端展示建议

- **四个都要**：  
  - 一级经销商、二级经销商：用 **`/system/user/list`**，传 `dealerLevel=1` 或 `2` 筛选；二级在表格中展示 `parentDistributorName` 作为从属关系。  
  - 终端客户/会员（有积分、会员码）：用 **`/mall/member/page`**，即为小程序用户中已绑定手机并关联到 ums_member 的记录。  
  - 仅小程序用户（未绑手机或未成会员）：同一接口 `/mall/member/page`，其中 `memberId/points/balance/level/memberCode` 可能为空。

- **二级经销商从属关系**：列表接口已返回 `parentDistributorId`、`parentDistributorName`，前端直接展示“所属一级：parentDistributorName”即可。

---

## 四、权限标识（需配置到菜单/按钮）

- 会员：`mall:member:index`、`mall:member:get`、`mall:member:add`、`mall:member:edit`、`mall:member:del`
- 用户列表/详情：`system:user:list`、`system:user:query`
