# 用户端与商家端功能排查清单（移动端已有功能）

本文档按「用户端（通用）」与「商家端」逐项对照现有后端接口与前端页面，标注 **✅ 已有** / **⚠ 部分有** / **❌ 待补充**，便于查漏补缺。

---

## 一、用户端（所有用户通用：经销商 + 终端客户）

### 1. 账户与安全

| 功能 | 后端接口 | 前端/说明 | 状态 |
|------|----------|-----------|------|
| 用户注册 | POST `/app/member/register`（手机号+验证码+密码） | 小程序以微信登录为主，C 端 H5 可走此接口 | ✅ 已有 |
| 登录 | 小程序：POST `/api/ma/wxuser/login`（code）；C 端：POST `/app/member/login`（手机+密码） | 小程序登录页、静默登录 | ✅ 已有 |
| 找回密码 | 无 | 需：短信验证码 + 新密码 重置 | ❌ 待补充 |
| 修改密码 | 后台：PUT `/system/user/profile/updatePwd`（sys_user）；C 端会员（ums_member）无 | 用户端修改密码需单独接口 | ❌ 待补充 |

**建议补充**：  
- C 端「找回密码」：如 POST `/app/member/reset-password`（phone、code、newPassword）。  
- C 端「修改密码」：如 PUT `/app/member/update-pwd`（oldPassword、newPassword），或 PUT `/api/ma/wxuser/password`（需先绑定手机且鉴权为当前用户）。

### 4. 积分与活动

| 功能 | 后端接口 | 前端/说明 | 状态 |
|------|----------|-----------|------|
| 每日签到领积分 | POST `/app/member/sign-in` | 我的-签到；接口需 Long memberId，小程序 token 为 openid 时可能失败 | ⚠ 部分有 |
| 推荐注册领奖励 | POST `/app/member/register` 传 `inviteCode`（推荐人会员码） | 注册页填推荐人；规则在 `tb_integral_rule.recommendIntegral` | ✅ 已有 |
| 积分红包 | POST `/app/member/send-packet`，规则 `redPacketSwitch` 控制 | 积分红包页；同上需 memberId 与手机验证 | ⚠ 部分有 |

**说明**：小程序当前通过 X-Wx-Token 解析为 openid，部分接口（如 `/app/member/*`）期望 `MemberUtils.getMemberId()` 为 ums_member 的 Long id，若未做 openid→memberId 映射会报错。建议签到、积分红包等增加 `/api/ma/*` 下接口，由 token 解析 openid→phone→ums_member 再调业务。

## 三、接口路径速查（用户端 / 商家端）

**用户端（小程序 / C 端）**  
- 登录/会员：`/api/ma/wxuser/login`、`/api/ma/wxuser/info`、`/api/ma/wxuser/member-code`  
- 地址：`/api/ma/useraddress/page`、POST、DELETE  
- 商品：`/api/ma/goodsspu/*`、`/api/ma/goodscategory/tree`、`/api/ma/home`  
- 订单：`/api/ma/orderinfo/*`  
- 签到/红包/注册：`/app/member/sign-in`、`/app/member/send-packet`、`/app/member/register`（注意 memberId 来源）  
- 优惠券/商品券：`/api/ma/wxuser/coupons`（小程序推荐）、`/app/coupon/my`  
- 抽奖：`/app/lottery/config`、`/app/lottery/draw`、`/app/lottery/record`  
- 公告：`/api/public/ma/notice/list`  

**商家端**  
- 扫码识别：GET `/api/mall/merchant/scan/user/{memberCode}`  
- 赠送积分：POST `/api/mall/merchant/scan/points`  
- 券核销：POST `/api/mall/merchant/scan/coupon/verify`  
- 注册二级经销商：POST `/registerDistributor`
