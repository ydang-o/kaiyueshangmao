# 订单超时自动取消（30 分钟）

## 实现方式

系统采用**两种方式**确保超时未支付订单被自动取消：

### 1. 定时任务（推荐，已默认开启）

- **类**：`OrderAutoCancelTask`
- **执行频率**：每 5 分钟一次
- **逻辑**：扫描「创建时间超过 30 分钟且未支付」的订单，调用取消逻辑（回滚库存、更新状态为已取消）
- **优点**：不依赖 Redis 配置，部署简单
- **配置**：已在 `DingyangmallApplication` 上开启 `@EnableScheduling`

### 2. Redis Key 过期监听（可选）

- **监听器**：`RedisKeyExpirationListener`
- **逻辑**：下单时在 Redis 写入 `mall:order:is_pay_0:{orderId}`，TTL 30 分钟；key 过期时触发监听器取消订单
- **前置条件**：Redis 需开启 key 过期事件

**Redis 配置**（redis.conf 或启动参数）：

```
notify-keyspace-events Ex
```

或通过 Redis CLI 临时设置：

```
CONFIG SET notify-keyspace-events Ex
```

> 若未配置，Redis 不会发布 key 过期事件，监听器不会触发；定时任务可弥补此情况。

## 超时时间

- 普通订单：30 分钟（`MallConstants.ORDER_TIME_OUT_0`）
- 秒杀订单：5 分钟（`MallConstants.ORDER_TIME_OUT_0_SECKILL`）

## 订单取消后

- 订单状态更新为「已取消」（status=5）
- 扣减的库存回滚
- 用户可在「我的订单」中看到已取消订单（可删除）
