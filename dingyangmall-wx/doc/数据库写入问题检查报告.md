# 数据库写入问题检查报告

## 一、当前方案：使用旧表 wx_user

小程序登录与更新用户信息已改为使用**旧表 `wx_user`**（项目主 SQL 中已存在），不再使用 `wx_ma_user`。

- **表名**：`wx_user`
- **字段映射**：openid→open_id，unionid→union_id，昵称→nick_name，头像→headimg_url，插入时 app_type='1' 表示小程序
- **主键**：`id`（插入新记录时用 IdUtils.simpleUUID() 生成）；按 `open_id` 唯一键做 ON DUPLICATE KEY UPDATE

若仍出现 `Table 'xxx.wx_user' doesn't exist`，请执行项目主 SQL（如 `sql/dingyangmall_ry_fixed.sql`）初始化库表。

---

## 二、写入逻辑检查（结论：正常）

### 1. 写入时机

| 场景 | 位置 | 说明 |
|------|------|------|
| 登录 | `WxUserApi.login()` | code2Session 成功后，调用 `wxMaUserMapper.upsert(u)` 写入 `wx_user` |
| 更新用户 | `WxUserApi.updateUser()` | 更新昵称/头像后，再次 `upsert` 到 `wx_user` |

### 2. Mapper 与实体

- **WxMaUserMapper.xml**：操作表 `wx_user`，`selectByOpenid` 按 `open_id` 查询，`upsert` 使用 `INSERT ... ON DUPLICATE KEY UPDATE`（唯一键 `open_id`）
- **WxMaUser 实体**：`id`（对应 wx_user.id）、`openid`→open_id、`unionid`→union_id、`nickname`→nick_name、`avatarUrl`→headimg_url
- **参数绑定**：`@Param("u")` 与 XML 中 `#{u.openid}` 等一致

### 3. 注入与空指针

- `wxMaUserMapper` 使用 `@Autowired` 注入，无 `required = false`
- 调用前有 `if (wxMaUserMapper != null)` 判断（防御性），正常部署下不会为 null

**写入逻辑本身没有问题，失败原因只有一个：表不存在。**

---

## 三、解决方案

当前已改为使用旧表 **wx_user**，无需新建表。若库中尚无该表：

1. 执行项目主 SQL（如 `sql/dingyangmall_ry_fixed.sql` 或 `docker/db/dingyangmall_ry.sql`），其中已包含 `wx_user` 表结构及数据。
2. 确认 `application-druid.yml` 中 jdbc url 的库名与执行 SQL 的库一致；当前用户具备 `INSERT`、`UPDATE`、`SELECT` 权限。

1. 小程序重新登录
2. 执行 `SELECT * FROM wx_user WHERE app_type = '1';`，应能看到小程序用户（open_id 为微信 openid，nick_name、headimg_url 为昵称与头像）
