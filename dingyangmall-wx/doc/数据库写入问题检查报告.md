# 数据库写入问题检查报告

## 一、根本原因

**错误信息**（来自 `sys-error.2026-02-24.log`）：

```text
java.sql.SQLSyntaxErrorException: Table 'dingyangmall_ry.wx_ma_user' doesn't exist
```

数据库中 **`wx_ma_user` 表不存在**，导致登录和更新用户信息时的 `upsert` 写入失败。

---

## 二、写入逻辑检查（结论：正常）

### 1. 写入时机

| 场景 | 位置 | 说明 |
|------|------|------|
| 登录 | `WxUserApi.login()` 第195-200行 | code2Session 成功后，调用 `wxMaUserMapper.upsert(u)` 将 openid/unionid 落库 |
| 更新用户 | `WxUserApi.updateUser()` 第411-418行 | 更新昵称/头像后，再次 `upsert` 到 `wx_ma_user` |

### 2. Mapper 与实体

- **WxMaUserMapper.xml**：`upsert` 使用 `INSERT ... ON DUPLICATE KEY UPDATE`，主键为 `openid`
- **WxMaUser 实体**：`openid`、`unionid`、`nickname`、`avatarUrl` 与表字段一一对应
- **参数绑定**：`@Param("u")` 与 XML 中 `#{u.openid}` 等一致

### 3. 注入与空指针

- `wxMaUserMapper` 使用 `@Autowired` 注入，无 `required = false`
- 调用前有 `if (wxMaUserMapper != null)` 判断（防御性），正常部署下不会为 null

**写入逻辑本身没有问题，失败原因只有一个：表不存在。**

---

## 三、解决方案

### 方案 A：执行建表 SQL（推荐）

建表脚本已存在：`d:\work_boss\dingyangMall\sql\wx_ma_user.sql`

**操作步骤：**

1. 在 MySQL 客户端或 Navicat 中连接 `dingyangmall_ry` 数据库
2. 执行以下 SQL（或直接执行 `sql/wx_ma_user.sql` 文件）：

```sql
USE dingyangmall_ry;

CREATE TABLE IF NOT EXISTS wx_ma_user (
  openid      VARCHAR(64)  NOT NULL COMMENT '小程序 openid（唯一）',
  unionid     VARCHAR(64)  NULL     COMMENT '开放平台 unionid（若有）',
  nickname    VARCHAR(128) NULL     COMMENT '昵称',
  avatar_url  VARCHAR(512) NULL     COMMENT '头像URL',
  create_time DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  update_time DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (openid),
  KEY idx_unionid (unionid)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='小程序用户信息';
```

3. 重启后端服务后，登录与更新用户信息应可正常写入

### 方案 B：表已存在但报错

若确认表已创建仍报错，请检查：

- `application-druid.yml` 中 `jdbc:mysql://.../dingyangmall_ry` 的库名是否正确
- 当前连接用户对该库是否有 `CREATE TABLE`、`INSERT`、`UPDATE` 权限
