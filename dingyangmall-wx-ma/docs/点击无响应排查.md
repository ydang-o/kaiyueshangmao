# 小程序点击事件无响应排查

## 已做配置与样式防护

1. **页面可点击**：`styles/theme.wxss` 中为 `page` 和 `.tm-page` 增加了 `pointer-events: auto`，避免被其他样式误覆盖导致整页无法点击。
2. **自定义 tabBar**：`custom-tab-bar/index.wxss` 中 `.tab-bar` 设为 `pointer-events: none`，仅 `.tab-item` 为 `pointer-events: auto`，避免整条 tab 栏挡住页面内容点击。
3. **广告位**：首页 `<ad>` 外包了一层 `.ad-wrap`，限制在文档流内，避免遮挡整页。
4. **按钮**：首页和「我的」页的主要按钮已加 `type="button"`、`hover-class="button-hover"`，避免误触发表单或默认样式影响点击。

## 请先确认这两点

### 1. 运行的是「编译后」的小程序

- 必须在 **HBuilderX** 里用 **运行 → 运行到小程序模拟器 → 微信开发者工具** 打开项目。
- 微信开发者工具里打开的目录必须是 **`unpackage/dist/dev/mp-weixin`**，不能是项目根目录。
- 若打开的是根目录，看到的是源码结构（如 `App.vue`、`pages/home/index.vue`），实际运行的可能不是 uni-app 编译结果，**所有 @tap 事件都不会生效**。

### 2. 微信开发者工具本地设置

- **详情 → 本地设置** 中勾选 **「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」**。
- 若未勾选，部分环境下可能影响页面行为或请求，间接导致异常。

## 若仍无响应

- 在 **Console** 看是否有报错（如 `__e is not a function`、或页面/组件报错），有则先解决报错再测点击。
- 临时关闭广告：在 `config/env.js` 中把 `adEnable` 改为 `false`，重新编译运行，看点击是否恢复，用于判断是否与广告组件有关。
