## 十、通用说明

- **请求头**：小程序请求会带 `app-id`、用户登录后带 `third-session`；商家端请求带 `Authorization: Bearer {merchantToken}`。
- **响应约定**：业务成功时建议统一返回 `code: 200`，失败时前端会弹 `res.data.msg`；`code: 60001` 时前端会清 session 并重新拉取当前页。
- **静态资源**：TabBar 图标、占位图等放在小程序包内 `static/`（如 `static/tabbar/`、`static/img/`），不请求后端，减轻服务端压力。

## 十一、前端静态资源（需打包，不走后端）

以下为小程序包内路径，需随包上传，不依赖后端接口：

| 路径 | 用途 |
|------|------|
| `static/tabbar/home.png` / `home-active.png` | 首页 Tab 未选/选中图标 |
| `static/tabbar/category.png` / `category-active.png` | 分类 Tab 未选/选中图标 |
| `static/tabbar/user.png` / `user-active.png` | 我的 Tab 未选/选中图标 |
| `static/img/no_pic.png` | 商品/订单等占位图 |
| `static/img/5-1.png` ~ `5-4.png`、`7-1.png` 等 | 个人中心宫格图标 |
| `static/img/shopping-cart.jpg` | 购物车空状态图 |

图标建议尺寸：TabBar 81×81 px，单张 ≤ 40KB。
