# UI 重设计审计清单（天猫系）

## 当前页面结构
- 页面总数：19（`pages/**/index.vue`）
- 公共组件：5（`components/**/index.vue`）
- 全局样式来源：`app.wxss` + ColorUI 工具类

## 主要审美与体验问题
- **视觉语言不统一**：颜色体系混用（蓝/橙/黑/红多套并行），按钮圆角与阴影风格不一致。
- **页面层级弱**：卡片、分区标题、间距规则在不同页面差异较大，信息密度不稳定。
- **转化路径不突出**：核心行为按钮（加购、下单、支付）在若干页面视觉权重不够一致。
- **状态页不统一**：空状态、加载状态、错误提示样式风格分散。
- **组件复用潜力未充分释放**：商品卡、订单操作、步进器已抽离，但样式层面未形成统一设计 token。

## 复用关系梳理
- 商品展示：`goods-card-index`、`goods-row` 被首页、列表、购物车推荐等多页复用。
- 订单操作：`order-operate` 被订单列表、订单详情复用。
- 数量控件：`base-stepper` 被购物车与下单相关流程复用。

## 改造策略
1. 先建立主题 token 与组件样式层（`styles/theme.wxss`、`styles/components.wxss`）。
2. 优先改造核心链路（首页→分类/列表→详情→购物车→确认下单）。
3. 再改订单与用户资产页面（订单、我的、地址、优惠券、积分、商家）。
4. 全站做统一收敛（按钮、卡片、标题、状态、间距、tab 导航、阴影）。
